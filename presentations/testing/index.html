Testing can seem like a bit of a drag, and if you're like me, you're sceptical.

Here's where I testify :)

Benefits of testing

  • Unit tests can be a powerful force for design
  • Produces testable code
  • With a test suite, reusability is built in
  • Tests can be aligned directly with ACs (BDD)
  • Test are the docs
  • Refactor with confidence

Problems

  • It does take more time
  • It can be a contraint (tests require grooming)
  • David Heinenmeier Hannson (DHH) - TDD is dead. Long live testing.
    • TDD fundamentalism is backwards/self-defeating/stupid.
    • We should be testing, but not feetishizing test first and unit tests.
    • Read: http://david.heinemeierhansson.com/2014/tdd-is-dead-long-live-testing.html

What should we be testing?

  • TDD is probably not viable for what we do.
  • Unit and integration tests might be appropriate:
    • Verifying particularly complex code/cases
    • When sending our children out into the world (i.e. Drupal.org)
    • Possibly as part of the process of fixing stuff (i.e. regression tests and validation)
  • Others? Frontend? CSS?
